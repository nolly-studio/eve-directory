On listings, **integrations** are short slugs (for example `slack`, `github`, `linear`) for channels and connections the agent is built around. They power `/integrations/[name]` filters and chips on cards.

## Official Eve docs

Eve publishes install/quick-start recipes at [eve.dev/integrations](https://eve.dev/integrations). Deep docs live under [Channels](https://eve.dev/docs/channels) and [Connections](https://eve.dev/docs/connections). This site does **not** fork those guides.

## After-install recipes (this site)

When you install a catalog agent, you still need to wire **your** Connect clients or env secrets. These short pages cover that handoff only:

- [Slack](/docs/integrations/slack) — swap the Connect UID in `agent/channels/slack.ts`
- [GitHub](/docs/integrations/github) — GitHub App env vars
- [Notion](/docs/integrations/notion) — `vercel connect create notion`
- [Telegram](/docs/integrations/telegram) — BotFather token + webhook
- [Linear](/docs/integrations/linear) — MCP Connect client (or Linear Agent channel)

See also [After you install](/docs/after-you-install).

## On this site

- Browse the [integrations directory](/integrations) or open `/integrations/<name>` for matching agents
